Overview

Onyx is a lightweight browser extension for saving and inserting reusable prompt snippets. It was built with a local-first mindset to eliminate backend complexity and protect user data. The design prioritizes speed and simplicity over feature bloat. It integrates with Chrome's popup interface and handles snippet persistence reliably. The extension is live and usable across websites and text fields.

Value
  • Quickly save and recall prompt snippets anywhere in the browser
  • Minimal UI that stays out of the way
  • Works fully offline/local — no cloud storage required
  • Fast insert into text fields with a popup and hotkeys
  • Simple add/edit/delete snippet management
  • Published on Chrome Web Store with low friction install

01.

Architecture

Local-first removes friction

  • Plug and play
    Add the extension and start saving prompts—no sign-up, no API keys, no set-up wizard.
  • Always available
    Prompts stay in the browser, so they work on airplanes, behind corporate firewalls, and offline.
  • Zero overhead
    No server to host, monitor, or secure. That means $0 monthly cost and no on-call pages at 3 a.m.
  • Fast store approval
    The only Chrome permission we request is storage. With no remote calls or analytics, review finished in six hours.
  • Built-in privacy
    Because data never leaves the device we can say, plainly, "Your prompts stay in your browser." Users trust that.

02.

User Experience

Small, focused UX wins

  • Loads in < 50 ms
    The popup is a single 28 KB script—no frameworks, no web fonts. It appears before the browser records a paint.
  • Keyboard-first flow
    Type x/ to open the picker, use arrows to highlight, press Enter to insert (or Shift + Enter on Claude). Esc closes it. Mouse is optional.
  • One screen, one job
    The popup shows a list and an add/edit form. A separate Options page repeats the same UI for bulk edits. No tabs, no buried settings.
  • Safe interactions
    Edit mode is clearly labeled and cancelable. Deletes are hidden until hover, with an "Undo" snackbar coming next.

Result: New users added a snippet in under 20 seconds, inserted it in chat in under 10, and then forgot the extension was running—that invisible speed is exactly what we aimed for.

03.

Technical Constraints

Respect host-site constraints - design graceful fall-backs

We spent a full afternoon trying to make Enter confirm a snippet on Claude.ai, exactly as it does on Gemini and ChatGPT. The steps we tried illustrate why "just intercept the key" is sometimes impossible:

Attempt Technique Outcome
Bubble-phase listener document.addEventListener('keydown', …) Fires after Claude's React handler; message already sent.
Capture-phase listener Same call with useCapture = true, plus e.stopImmediatePropagation() Claude still sees the event—its input sits in a shadow DOM with its own capture listeners that run first.
Shadow-DOM retargeting Attach listener to shadowRoot.host Claude re-creates the host node on every render; listener is lost.

Why we stopped forcing Enter on Claude

  • Too risky – If our code ever misses the key, Claude still fires Send and the user ships an unfinished message.
  • Brittle – Claude's DOM and event order change without warning; we would chase bugs forever.
  • Wasteful – Re-attaching listeners on every render slows things down for everyone.

Simple fall-backs that work everywhere

  • Shift + Enter – Claude treats this as "new line," so we can safely use it as Confirm.
  • Tab – A natural "select" key in autocomplete lists and never tied to Send.

Rule we shipped

Picker open:

Enter → confirm (all sites except Claude)
Shift + Enter → confirm (all sites)
Tab → confirm (all sites)

Picker closed:

Page handles keys as usual

Accepting one extra keystroke on Claude gave us a solution that is safe, future-proof, and free of hacky DOM surgery.
